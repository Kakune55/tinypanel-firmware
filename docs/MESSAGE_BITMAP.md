# 消息位图格式

消息正文可使用 `[[BMP:...]]` 标签嵌入单色位图图像。

## 标签

```text
[[BMP:<hex>]]
```

有效载荷为十六进制文本。有效载荷内的空白字符将被忽略。

## 格式

```text
FF<width><height><bitmap-data>
```

- `FF` 为位图有效载荷标记符。
- `width` 为 1 字节。
- `height` 为 1 字节。
- 两者均以十六进制编码。

示例：`FFC840` 表示 `200x64`。

## 像素数据

- 1 位（bit）代表 1 个像素。
- 像素按行优先顺序排列：从左到右，从上到下。
- 每个字节内的位按从高到低的顺序读取。
- `1` 表示黑色像素。
- `0` 表示白色像素。

数据长度至少需满足：

```text
ceil(width * height / 8) 字节
```

## 示例

8x8 全黑图像：

```text
[[BMP:FF0808FFFFFFFFFFFFFFFF]]
```

8x8 对角线图像：

```text
[[BMP:FF08088040201008040201]]
```

200x64 图像框架：

```text
[[BMP:FFC840<1600 字节的位图数据（即 3200 个十六进制字符）>]]
```

图像将独占一行渲染，并在消息正文区域内居中显示。
